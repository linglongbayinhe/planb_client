# 微信开发者工具常见警告说明

在「运行到小程序模拟器 -> 微信开发者工具」时，控制台可能出现以下提示。本文说明原因及可采取的措施。

---

## 1. [Deprecation] SharedArrayBuffer will require cross-origin isolation as of M92...

**含义**：Chrome M92 起，使用 SharedArrayBuffer 的页面需处于跨域隔离环境，否则会给出弃用警告。

**来源**：模拟器基于 Chromium，运行时会触发该警告；项目源码未直接使用 SharedArrayBuffer，多为框架或开发者工具内部使用。

**能否在项目里消除**：

- **微信开发者工具模拟器**：不能。模拟器由工具自带 Chromium 提供，无法在项目中为其设置跨域隔离响应头，该警告来自运行环境本身。
- **运行到浏览器（H5）**：已在项目根目录通过 `vite.config.js` 为开发服务器配置了 COOP/COEP 响应头，在浏览器中运行可避免或减轻该警告。

**建议**：模拟器中的该提示可忽略，不影响真机小程序运行；若需无警告的 H5 调试，请使用「运行到浏览器」。

---

## 2. 配置中关闭合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书检查 / 工具未校验合法域名...

**含义**：当前 微信开发者工具 的「本地设置」中勾选了「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」，因此工具不会对请求域名、TLS、证书做校验。

**来源**：  
- 工具端：详情 → 本地设置 → 「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」被勾选。  
- 项目端：`manifest.json` 中 `mp-weixin.setting.urlCheck: false`，表示小程序运行时也不做域名校验（便于本地开发请求云函数等）。

**能否在项目里消除**：

- 该**控制台文案**由 uni-app 运行时在检测到工具未校验时输出，无法在项目代码中关闭。
- 若希望控制台不再出现该提示，需要：
  1. 在 微信开发者工具 中取消勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」；
  2. 在 微信公众平台 → 开发管理 → 开发设置 中配置好 **request 合法域名**（如云函数域名等），否则请求会报错。

**建议**：开发阶段可保持当前设置，该警告不影响功能；**正式发布前**请在公众平台配置好合法域名，并视情况将 `manifest.json` 中 `urlCheck` 设为 `true` 做一次完整校验测试。

---

## 3. 文章推荐 / getSystemInfo API 提示 / HarmonyOS 平台

**含义**：开发者工具推送的说明——基础库 3.7.0 起支持 HarmonyOS，建议用 `wx.getDeviceInfo()` 做平台判断以更好兼容 HarmonyOS。

**来源**：工具内的信息推荐，非项目代码报错。

**项目侧**：uni-app 运行时已在使用 `wx.getDeviceInfo()`（可用时）并回退到 `getSystemInfoSync()`，无需在业务代码中额外修改。

**建议**：可视为说明类提示，无需处理；若要兼容 HarmonyOS，可参考微信官方文档对 `wx.getDeviceInfo()` 的说明。

---

## 总结

| 警告 | 能否在项目内消除 | 建议 |
|------|------------------|------|
| SharedArrayBuffer 弃用 | 模拟器环境不能；H5 已通过 vite 配置处理 | 模拟器可忽略；H5 用「运行到浏览器」 |
| 未校验合法域名 / TLS / 证书 | 不能关闭该日志；可改工具设置与公众平台配置 | 开发可保持现状；上线前配置合法域名并做校验测试 |
| HarmonyOS / getDeviceInfo 推荐 | 无需改代码 | 仅作参考，框架已做兼容 |
