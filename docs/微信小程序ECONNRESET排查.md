# 微信小程序运行时报 ECONNRESET 排查指南

## 错误含义

- **code: ECONNRESET**：连接被对端或网络在中途重置。
- **Client network socket disconnected before secure TLS connection was established**：在 TLS 握手完成前，客户端连接就被断开。

**发生时机**：HBuilderX 编译完成后，会通过 **HTTPS** 连接本机的**微信开发者工具**提供的“服务端口”，用于自动打开项目、刷新等。若该连接在建立 TLS 前被断开，就会报上述错误。

因此问题通常出在：**HBuilderX ↔ 微信开发者工具** 这一段本地通信，而不是外网或 uniCloud。

---

## 一、优先检查：微信开发者工具「服务端口」

HBuilderX 必须能连上微信开发者工具开启的“服务端口”，否则就会 ECONNRESET。

### 1. 开启服务端口

1. 先**单独打开微信开发者工具**（不要通过 HBuilderX 启动）。
2. 右上角 **「设置」** → **「安全设置」**（或 **「安全」**）。
3. 勾选 **「服务端口」**（或「开启服务端口」），并记住端口号（默认多为 **80** 或 **20xxx**）。
4. 若没有该选项，在 **「设置」→「通用设置」** 中查找与「服务端口」「命令行」相关的开关并打开。
5. 保存后**完全关闭微信开发者工具再重新打开**，确保端口生效。

### 2. 确认 HBuilderX 端口一致（若可配置）

- 在 HBuilderX 中：**运行 → 运行到小程序模拟器 → 运行设置**（或 工具 → 设置 → 运行配置）中，若有「微信开发者工具端口」或「命令行端口」，请填与微信开发者工具中**相同的端口**。
- 若无法配置，则保持微信开发者工具使用默认端口即可。

---

## 二、排查步骤（按顺序做）

### 步骤 1：保证「先开微信开发者工具，再运行」

1. **先** 手动打开微信开发者工具，并已开启「服务端口」。
2. 在 HBuilderX 中再点击 **运行 → 运行到小程序模拟器 → 微信开发者工具**。

若这样能正常打开项目，说明之前是 HBuilderX 在工具未就绪时就去连端口导致 ECONNRESET。

---

### 步骤 2：检查端口是否被占用

微信开发者工具服务端口多为 **80** 或 **20xxx**（以设置里显示为准）。

**Windows 查看占用：**

```bat
netstat -ano | findstr "80"
```

或（若端口为 20482）：

```bat
netstat -ano | findstr "20482"
```

若该端口被其他程序占用，可：

- 关闭占用该端口的程序，或
- 在微信开发者工具中更换为其它未被占用的端口，并确保 HBuilderX 中配置的端口一致（若支持配置）。

---

### 步骤 3：关闭代理 / VPN / 网络加速

- 代理或 VPN 有时会拦截或改写本机 **localhost / 127.0.0.1** 的 HTTPS，导致 TLS 未建立就断连。
- 临时关闭系统代理、VPN、网络加速器后，再在 HBuilderX 中重新「运行到微信开发者工具」试一次。

若使用 **Clash / V2Ray 等**：在规则中把 `127.0.0.1`、`localhost` 设为直连（不走代理）。

---

### 步骤 4：防火墙 / 安全软件

- 临时关闭 Windows 防火墙或安全软件，再次运行，看是否仍报 ECONNRESET。
- 若关闭后正常，再在防火墙中为 **HBuilderX** 和 **微信开发者工具** 添加允许入站/出站规则，避免长期关防火墙。

---

### 步骤 5：以管理员身份运行（可选）

- 右键 **HBuilderX** → **以管理员身份运行**。
- 右键 **微信开发者工具** → **以管理员身份运行**。
- 再次在 HBuilderX 中运行到微信开发者工具。

部分环境下权限不足会导致本地端口或 TLS 行为异常。

---

### 步骤 6：重置 / 重装

**仅在前述都无效时考虑：**

1. **微信开发者工具**  
   - 完全卸载后重装，或  
   - 在设置中恢复默认设置，再重新开启「服务端口」。

2. **HBuilderX**  
   - 检查是否有「运行到小程序」相关更新，或重新安装 HBuilderX。

---

## 三、替代方案：不依赖自动连接，手动导入项目

若 ECONNRESET 无法解决，可以**不用 HBuilderX 自动打开微信开发者工具**，改为手动导入编译结果：

1. 在 HBuilderX 中照常点击 **运行 → 运行到小程序模拟器 → 微信开发者工具**，**只做编译**（即便报 ECONNRESET，编译通常已经完成）。
2. 打开本机目录：  
   **`D:\Projects\HBuilderProjects\planb_client\unpackage\dist\dev\mp-weixin`**
3. 打开**微信开发者工具** → **导入项目** → 选择上述 **`mp-weixin`** 目录。
4. 项目 AppID 选择你的小程序 AppID（或测试号）。
5. 之后每次在 HBuilderX 中修改代码并再次「运行到微信开发者工具」后，到微信开发者工具里点击 **「编译」** 或 **「重新编译」** 即可看到最新效果。

这样完全不依赖 HBuilderX 与微信开发者工具之间的 TLS 连接，可规避 ECONNRESET。

---

## 四、简要对照表

| 现象           | 优先检查                         |
|----------------|----------------------------------|
| 一运行就 ECONNRESET | 微信开发者工具「服务端口」是否开启 |
| 端口已开仍报错 | 先开微信开发者工具，再在 HBuilderX 运行 |
| 仍报错         | 关代理/VPN、查端口占用、关防火墙试一次 |
| 仍无法解决     | 用手动导入 `unpackage/dist/dev/mp-weixin` 方式运行 |
