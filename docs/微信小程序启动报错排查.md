# 微信小程序启动报错「[] [object Object]」排查说明

## 错误现象

控制台仅显示一行：
```text
[] [object Object](env: Windows,mp,2.01.2510280; lib: 3.14.2)
```

说明某处抛出的**错误对象被直接转成字符串**，得到 `[object Object]`，真实错误信息被吞掉。

---

## 可能原因（按常见度）

### 1. 使用 SVG 作为 `<image>` 的 src（高概率）

- **原因**：微信小程序对 `<image src="*.svg">` 支持有限，部分基础库或 **Skyline 渲染**下会报错或导致异常。
- **表现**：启动或首屏渲染时即报错，且错误信息不完整。
- **处理**：底部 Tab、列表等使用的图标改为 **PNG**（或其它小程序支持的图片格式），不要用 SVG。

### 2. pages.json 配置问题

- 检查 `pages` 第一项是否为入口页，路径是否真实存在（如 `pages/index/index` 对应 `pages/index/index.vue`）。
- 检查是否有语法错误（多余逗号、引号不匹配等）：用 JSON 校验工具或 HBuilderX 检查。

### 3. 入口页或全局组件报错

- 入口页（如 `pages/index/index.vue`）或其中引用的组件在**创建/渲染阶段**抛错，会被包装成上述格式。
- 检查：入口页及它引用的 `HomeView`、`CluesView`、`SendView`、`SettingsView` 是否有在 `data`/`created`/模板里访问不存在的属性、未注册的组件等。

### 4. uniCloud 未关联或初始化异常

- 若项目使用了 uniCloud，但未在 HBuilderX 中**关联云服务空间**，或运行到微信开发者工具时网络/配置异常，可能触发错误。
- 处理：HBuilderX 中右键 `uniCloud` 目录 → 关联云服务空间；若暂时不用云，可先注释掉调用 uniCloud 的代码验证是否与启动报错有关。

### 5. manifest.json / 微信小程序配置

- 确认 `mp-weixin` 下的 `appid` 与微信公众平台一致。
- 确认没有不可见字符或 JSON 语法错误。

---

## 建议排查步骤

1. **先看更详细错误**
   - 在**微信开发者工具**中：打开「调试器」→ Console，看是否有红色报错或堆栈。
   - 在 HBuilderX：菜单「运行」→「运行配置」→ 看是否可勾选「控制台详细日志」或类似选项。
   - 在入口（如 `main.js` 或 `App.vue` 的 `onLaunch`）最顶部加：
     ```js
     try {
       // 原有逻辑
     } catch (e) {
       console.error('startup error', e && e.message, e && e.stack)
     }
     ```
     有时能把真实错误打出来。

2. **优先排除 SVG**
   - 将底部 Tab 等使用 `.svg` 的 `<image>` 全部改为 `.png`（并确保对应 PNG 文件存在），重新运行到微信小程序。若错误消失，即可确认与 SVG 有关。

3. **最小化复现**
   - 临时新建一个仅含单页、无复杂组件的 `pages.json` 入口，看是否仍报错。若不再报错，再逐步恢复原入口和组件，定位到具体页面或组件。

4. **检查最近修改**
   - 若最近改过 `pages.json`、`manifest.json`、入口页或 Tab 栏，优先回退或按上面几点逐项检查。

---

## 本项目已做修改

- 已将**底部 Tab 栏**的图标路径由 `.svg` 改为 `.png`，以避免微信小程序下 SVG 导致的启动/渲染异常。
- 当前已临时使用 `play.png` / `pause.png` 作为所有 Tab 的图标，保证无需新增文件即可启动。若需恢复为独立图标，请将 `pages/index/index.vue` 中 `tabs` 改为对应 PNG 路径，并确保以下文件存在（可从原 SVG 导出为 PNG）：
  - `static/icons/home.png`、`home-active.png`
  - `static/icons/clues.png`、`clues-active.png`
  - `static/icons/send.png`、`send-active.png`
  - `static/icons/settings.png`、`settings-active.png`
