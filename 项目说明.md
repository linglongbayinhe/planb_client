# Plan B 客户端 — 项目说明

## 一、项目概览

- **项目名**：`planb_client`（后手 / Plan B 客户端）
- **一句话**：多端（含微信小程序）应用，用于管理「线索库」、配置「计划发送」（收件人、触发时间等），到期后由云函数发邮件通知。
- **技术栈**：
  - 前端：**uni-app**（Vue 3），可编译到微信小程序等
  - 后端：**uniCloud 阿里云**（云函数/云对象 + 云数据库）
  - 小程序 AppID：在 `manifest.json` 的 `mp-weixin.appid`（当前为 `wx8e293e6f73b4fe8b`）

---

## 二、功能模块与入口

| 模块       | 入口/文件                                      | 功能简述                                           |
|------------|-----------------------------------------------|----------------------------------------------------|
| **首页**   | `pages/index/index.vue` → `components/HomeView.vue` | 单页 Tab：首页 / 线索库 / 发送 / 设置；首页展示倒计时、快捷入口 |
| **线索库** | `components/CluesView.vue`                    | 线索的增删改查、分类、导出 PDF 等                   |
| **发送设置** | `components/SendView.vue`                   | 收件邮箱/手机、展示姓名、通知正文、触发周期、开启/暂停、同步到云 |
| **设置**   | `components/SettingsView.vue`                | 主题、语言、登录/注册等                            |
| **登录/注册** | `components/sheets/LoginRegisterSheet.vue` | 微信/QQ 授权登录，调用云对象            |

**状态与持久化**：前端状态与本地持久化在 `store/index.js`（Vue 3 reactive + `uni.setStorageSync`），键包括：`clues`、`plan`、`user`、`theme`、`language`、`clueCategoryOrder` 等。

### 3.2 云开发（uniCloud 阿里云）

| 用途                   | 位置                              |
|------------------------|-----------------------------------|

### 3.3 登录 / 鉴权 / 邮件（敏感或易错配置）

| 配置                                   | 位置                                                                                                                                 | 说明 |
|----------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------|------|
| 微信 appid/appsecret、密码加密密钥、**SMTP 发信**、token 等 | **`uniCloud-aliyun/cloudfunctions/common/uni-config-center/uni-id/config.json`**（唯一一份） | 仓库内仅保留占位/空值。**真实密钥**请在 **阿里云 函数计算 FC 控制台** 为对应云函数配置环境变量（WECHAT_MP_APPID、WECHAT_MP_APPSECRET、SMTP_HOST、SMTP_USER、SMTP_PASS、SMTP_FROM、UNI_ID_PASSWORD_SECRET 等），或在上传前在本地该 config 中填写并**勿提交**。代码优先读 `process.env`，再回退 config。 |
| 云函数依赖与公共模块                      | 各云函数 **`package.json`**。上传前请先右键 **`common/uni-config-center`** →「上传公共模块」；再对 **auth_provider、plan_send_check** 分别右键 →「管理公共模块依赖」→ 勾选 **uni-config-center**。若提示「公共模块存在同名冲突」，请在 HBuilderX 的 `common` 下**删除带绿色箭头的 uni-config-center**（链接/引用），只保留实体文件夹。 |      |

### 3.4 云函数 / 云对象清单与作用

| 名称                   | 类型   | 作用                                       | 前端调用位置（参考）     |
|------------------------|--------|--------------------------------------------|--------------------------|
| **auth_provider**      | 云对象 | 微信/抖音/小红书授权登录，写 uni-id-users 表 | `LoginRegisterSheet.vue`、`pages/index/index.vue` |
| **user_profile**       | 云对象 | 获取用户信息与发送计划配置（getBootstrap，返回 plan：enabled/sendDate/emails/phones/displayName/customGuide） | 同上 |
| **send_plan**         | 云对象 | 统一同步发送设置到 uni-id-users（emails/phones/displayName/customGuide/enabled/sendDate） | `SendView.vue`、`HomeView.vue` |
| **plan_send_check**    | 云函数 | 定时消费 `plan_send_tasks` 队列→发计划邮件/短信→按结果清理 `send_time` | 由**定时触发器**调用，不直接由前端调用 |
**迁移期兼容**：`send_email`、`send_phone`、`send_message`、`send_time`、`set_enable_sending` 仍保留为薄包装，内部统一转调 `send_plan`；前端主入口已切换到 `send_plan`。

**定时任务**：`plan_send_check/package.json` 的 **`cloudfunction-config.triggers`** 中配置，当前为每天一次执行（cron：`0 0 0 */1 * * *`），`timeout` 为 `600` 秒。

### 3.5 云数据库（uni-id-users 表 + 任务表）

| 内容                           | 位置 |
|--------------------------------|------|
| uni-id-users 表结构、权限、业务字段说明 | **`uniCloud-aliyun/database/uni-id-users.schema.json`** |
| plan_send_tasks 任务表结构与索引 | **`uniCloud-aliyun/database/plan_send_tasks.schema.json`**、**`uniCloud-aliyun/database/plan_send_tasks.index.json`** |
| users 表（已迁移，可作备份）Schema 说明  | **uniCloud-aliyun/database/users.schema.json**、**users-schema-说明.md** |
| plan_send_tasks 任务表字段说明 | **`uniCloud-aliyun/database/plan_send_tasks.schema.json`**、**`uniCloud-aliyun/database/plan_send_tasks-schema-说明.md`** |
| Schema 上传失败排查 | **`uniCloud-aliyun/database/Schema上传排查说明.md`** |

**uni-id-users 表与发送相关的业务字段**：`send_emails`、`send_phones`、`enable_sending`、`send_time`、`send_display_name`、`send_message` 等；统一提交入口为 `send_plan`。`plan_send_tasks` 用于存放待发送/待重试的轻量任务，部署时需先创建集合并上传复合索引 `{ status: 1, next_retry_at: 1 }`。
